Project Proposal
1. Project Overview

This project proposal aims to introduce the formation of an Agile team that will work collaboratively throughout the course. The proposal defines the team structure, leadership, and overall approach to teamwork in preparation for future project development phases.

2. Problem Statement

In team-based academic projects, lack of clear leadership, undefined roles, and weak coordination can lead to miscommunication and inefficiency. This proposal addresses the need for a clearly structured team with defined responsibilities to ensure effective collaboration and successful project execution.

3. Target Users
Primary User Type 1: Team Leader

Description: The person responsible for leading and coordinating the team.

Goals:

Organize team tasks and responsibilities

Communicate with the instructor

Ensure smooth team collaboration

Pain Points:

Managing time and coordination

Ensuring all members contribute equally

Primary User Type 2: Team Member

Description: Team members who contribute to the project tasks.

Goals:

Complete assigned tasks efficiently

Collaborate with the team leader

Gain practical experience

Pain Points:

Unclear task assignments

Lack of communication

4. Core Features (Must-Have – MVP)
1. Team Leadership

Description: Assign a clear team leader to manage and guide the team.

User Benefit: Improves coordination and decision-making.

2. Role Definition

Description: Clearly define roles and responsibilities for each team member.

User Benefit: Reduces confusion and overlapping tasks.

3. Communication Process

Description: Establish regular communication between team members.

User Benefit: Ensures transparency and quick issue resolution.

4. Task Coordination

Description: Organize and track tasks within the team.

User Benefit: Helps ensure timely task completion.

5. Additional Features (Nice-to-Have)

Regular retrospective meetings

Use of collaboration tools for documentation

Peer feedback and review
